# Site Pets Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a site-wide, theme-aware pet overlay to the portfolio with header controls, saved preferences, and click reactions, using `webpets/` as the implementation reference rather than a runtime dependency. Ship the first version with `crab/red`, `deno/green`, `fox/red`, `vampire/girl`, `dog/brown`, and `panda/black` as the initial preset pets.

**Architecture:** Port the small GIF-based pet engine into the Vite app as a dedicated overlay mounted from `App.tsx`. Split the work into theme wiring, persisted preference state, curated pet data, animation/rendering, and compact navigation controls so the animation loop stays isolated from UI state. Extend the reference behavior with an explicit click-reaction override and a data model that allows more pets to be added later without changing the engine.

**Tech Stack:** React 19, TypeScript, Framer Motion, Vite, Vitest, Testing Library
